для интерфейса использован gui фреймворк для go gioui

приложение оптимизировано, все расчеты вынесены в кеш, чтоб не нагружать cpu при отрисовке
логика расчета координат проиллюстрирована на изображении logics.png
- вычисляется координата точки пересечения
- исдет поиск максимального смещения по оси
- по смещению выбирается ось и ее середина
те алгоритм перевода координат смремится нарисовать точку
максимально близко к середине четверти в которой она расположена

для работы с ui используется контест (паттерн Context Object)
учитывая что пользователь не смогет работать со скоростью света, многопоточностью можно пренебречь
поэтому блокировки закомментированы

поддерживаются как крассические прямые kx + b где k это tg угла наклона, а b смещение
так и y = b где коэффициен k отстутствует

данные задаются в формате ключ=значение
формы ввода нет, тк по условиям задачи упор был на работу с fs

для запуска требуется:
- sdk golang
- загрузить зависимости через go mod tidy
- запустить сборку go build -ldflags="-H windowsgui" или через файл build.bat для windows

exe собран под win64
